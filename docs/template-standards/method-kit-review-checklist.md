# Method Kit Review Checklist

## Purpose

Use this checklist before accepting a PowerPoint template as an official QCC method kit.
The checklist applies to method guides, PowerPoint templates, source notes, and catalog entries.

## Required Content

| Check | Pass condition | Failure condition |
|---|---|---|
| Method identity | Method ID, method name, and QCC stage fit are clear and match the catalog. | Missing QCC stage fit or mismatched method identity. |
| Method logic | The template teaches the actual QCC method logic and preserves method-specific cautions. | generic template assets without QCC method context. |
| Demo labeling | Completed examples are labeled `DEMO EXAMPLE - not project evidence`. | demo data presented as project evidence. |
| Blank working surface | A blank copyable slide, worksheet, diagram, or chart is visibly separate from the demo. | Users must overwrite the demo without a blank project surface. |
| Inputs | Required data, worksheet fields, categories, or project context are visible. | Required inputs are missing or ambiguous. |
| Interpretation | Interpretation patterns help the user write a conclusion without overstating the method. | The slide invites unsupported root-cause or final-evidence claims. |
| Common mistakes | common mistakes and review risks are documented. | Known misuse is not warned against. |
| Facilitator checklist | A facilitator checklist provides pass/fail or equivalent review criteria. | Review criteria are absent or only decorative. |
| Python assist decision | The kit explains when PowerPoint is enough and when Python assist is recommended. | Python is required unnecessarily or never recommended for high-rigor evidence. |
| Evidence/source note | Source, date range, filters, assumptions, and calculation notes are prompted where relevant. | missing data source or missing date range on a data-dependent slide. |

## Evidence Failures

Treat these as review failures until corrected:

- demo data presented as project evidence;
- missing data source;
- missing date range;
- overwritten or undocumented formula cells in a chart method;
- unsupported final conclusion from a manual chart;
- high-risk data-dependent claim without reproducible evidence or a validated analysis path.

## Incoming Template Decision

Incoming templates stay in `templates/incoming/` and are not official until they pass this checklist.
Templates with private data, hidden notes, unsupported formulas, unknown method, unknown QCC stage, or unknown owner remain source assets.
