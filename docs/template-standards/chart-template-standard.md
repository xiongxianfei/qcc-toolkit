# Chart Template Standard

## Purpose

Chart templates are QCC decision tools, not decorative figures.
Every official chart method kit should help the user edit the chart correctly, interpret the pattern safely, record evidence context, and decide when Python assist is required.

## Required Chart Surfaces

| Surface | Required content |
|---|---|
| Chart decision guide | Decision supported, Pattern to look for, Safe conclusion, Overclaim to avoid, and next method or action. |
| Chart variant library | At least one common chart variant and guidance for when to use it. |
| Chart quality checklist | Source, Date range, Filters, assumptions, labels, units, sorting or binning rule, Formula check, and reviewer decision. |
| Sample data table | Required fields, example rows, and formula fields where relevant. |
| Blank project chart slide | Chart area, source/date/filter fields, key finding, next action, prepared-by/date. |
| Python assist trigger | Raw data, repeated generation, validation-heavy formulas, high-rigor evidence, large data, or final reviewed evidence. |

## Pareto-Specific Standard

A Pareto chart kit should include:

- simple count Pareto for draft focus decisions;
- Cumulative Pareto when the team needs top-share or threshold language;
- Before/after Pareto comparison when improvement impact is being discussed;
- Focus annotation when reviewers need to see the selected vital few;
- Percent and Cumulative percent fields when the template uses share or threshold statements;
- Formula check guidance so cumulative or percentage cells are not overwritten.

## Review Failures

Treat these as chart-template review failures:

- chart has no visible Source or Date range;
- filters or assumptions are missing when data were narrowed;
- labels, units, sorting, bins, or category definitions are ambiguous;
- formula cells are overwritten or undocumented;
- chart conclusion claims root cause without supporting analysis;
- manual PowerPoint chart is treated as authoritative high-risk evidence without Python assist or another validated path.
