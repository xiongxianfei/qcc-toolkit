# CR-M4-001: Catalog accepts mismatched method guide ownership

- Finding ID: CR-M4-001
- Severity: major
- Location: `qcc_toolkit/templates/__init__.py:76-123`; `tests/test_template_catalog_failures.py:8-110`
- Evidence: T13 requires mutated catalog fixtures with mismatched guide method IDs to fail validation.
  R48 requires automated checks for method IDs, template IDs, catalog paths, guide paths, script paths, and example project paths to be consistent.
  The current validator checks referenced paths, duplicate template IDs, and duplicate method ownership, but it never reads the referenced Markdown guide front matter to confirm the guide's `method_id` matches the catalog entry's `method_id`.
  A reviewer mutation changed a Pareto catalog entry to reference `docs/methods/check_sheet.md`; `validate_template_catalog()` accepted the catalog instead of failing.
- Required outcome: Catalog validation must fail when a template entry's `markdown_guide` front matter declares a different `method_id` than the entry.
  The M4 negative tests must include this mismatched guide method ID case.
- Safe resolution path: Parse the referenced Markdown guide front matter during catalog validation, compare its `method_id` with the catalog entry `method_id`, raise `CatalogValidationError` with entry and path details on mismatch, and add a failing-first test to `tests/test_template_catalog_failures.py`.
- needs-decision rationale: none
