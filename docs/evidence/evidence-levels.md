# Evidence Levels

## Purpose

Evidence levels define how much support a QCC chart, worksheet, or teaching visual needs before it is used.
They help teams distinguish conceptual visuals, drafts, project-presentation charts, formal-review charts, and audit or high-risk evidence.

## Levels

| Level | Use case | Evidence requirement |
|---|---|---|
| E0 - Concept | Training image, conceptual diagram, or method explanation visual. | Mark as conceptual only. No source data is required because the artifact is not project evidence. |
| E1 - Draft | Team discussion, rough worksheet, or learning exercise. | Record source data, date range, and scope or filters when available. Mark as draft. |
| E2 - Project presentation | Normal QCC project story or internal project review. | Preserve source data, date range, scope or filters, method, reviewer checklist, and review status. |
| E3 - Formal review | Management review, competition review, customer-facing review, or high-visibility decision. | Preserve calculation table, chart source, method guide version, reviewer status, assumptions, and exclusions. |
| E4 - Audit or high-risk | Safety, regulatory, customer-impact, or audit-sensitive evidence. | Use a validated analysis path or independent verification and preserve a full reproducibility record. |

## Required distinctions

- E0 visuals are conceptual only and must not be confused with final quantitative evidence.
- E1 drafts may be incomplete, but their limitations should be visible.
- E2 charts need enough evidence for ordinary project review.
- E3 charts need traceable calculations and method-version context.
- E4 charts need reproducibility and independent confidence, not only attractive formatting.

## Privacy note

Evidence notes should capture source context without exposing unnecessary private raw rows.
Use approved synthetic data for repository examples unless a data owner explicitly approves another dataset for publication.
