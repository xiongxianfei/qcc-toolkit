# Tool Selection Guidance

## Purpose

This guide helps users choose a suitable class of tool without making any named product the identity of QCC Toolkit.
First-slice method guidance uses tool-class guidance only.
Named-tool recipes are deferred unless later user testing proves a specific path is necessary.

## Tool classes

| Method or chart need | Suitable tool class | Guidance |
|---|---|---|
| Simple categorical chart | spreadsheet tools or charting tools | Use when data is small, calculations are transparent, and visual control matters. |
| Simple trend or before/after chart | spreadsheet tools, charting tools, or presentation tools | Use when users can inspect the source table and axis choices. |
| Root-cause diagram or worksheet | presentation tools, whiteboard tools, or diagramming tools | Use when flexible team editing and discussion matter. |
| Control chart or capability study | statistical tools or validated analysis tools | Use when limits, subgroups, assumptions, and rule interpretation require stronger review. |
| Large raw-data analysis | data-analysis tools or statistical tools | Use when cleaning, grouping, repeatability, and validation matter. |
| Formal reviewed evidence | any suitable tool class with preserved evidence | Require source data, calculation table when applicable, assumptions, and reviewer status. |

## First-slice rule

The first slice should name tool classes, not named products.
If a future guide needs a named-tool recipe, the relevant spec should record user-test evidence that the generic tool-class guidance was not enough.

## Review questions

- Does the guide explain what kind of tool is suitable?
- Does the guide avoid requiring a named product?
- Does the guide preserve source data, assumptions, and review status regardless of tool class?
- Does the guide recommend stronger validation for high-rigor evidence needs?
