# Check Sheet

## Legacy optional-aid note

This guide belongs to the earlier first-slice method-guide surface.
Under the current Markdown-first direction, official method kits live under `method-kits/`.
PowerPoint templates and Python aids are optional execution aids and do not override method-kit guidance.

## Summary

A Check Sheet is a structured tally form for collecting QCC observations consistently.
In the first slice it is a template-guided method.

## Purpose

A Check Sheet gives the team a consistent way to collect observations before analysis.
It makes what was counted, where it was counted, and when it was counted reviewable.

## QCC stage fit

Use it during Understand Current Condition to collect baseline facts before analysis.

## When to use

Use a Check Sheet when the team needs repeatable counts by category, location, shift, time period, or other agreed stratification.

## When not to use

Do not use it when the observation definition is unclear or when team members cannot collect data consistently.

## Inputs required

Inputs include the observation definition, category list, collection period, collector, and tally area.

## Outputs produced

The method produces a completed tally worksheet that can feed Pareto, Histogram, or Stratification later.

## Procedure

1. Define what counts as one observation.
2. Define categories and collection context.
3. Tally observations during the agreed period.
4. Review totals and collection problems.
5. Convert validated tallies into data for later analysis when needed.

## PowerPoint template workflow

1. Open the Check Sheet PowerPoint method template.
2. Review the completed DEMO EXAMPLE - not project evidence slide.
3. Replace the check items, tallies, and review notes with project facts.
4. Copy the blank working slide or worksheet into the project deck.
5. Record the source, date range, collection rules, filters, and prepared-by/date.
6. Use the facilitator checklist before using totals in later charts.

PowerPoint is enough for teaching, draft, and small manual tally sheets.

## Blank working slide or worksheet

The blank working slide or worksheet should include check items, tally columns, totals, source, date range, collection rules, key conclusion, next action, and prepared-by/date.
Keep categories stable during collection and document any exclusions.

## Interpretation guidance

Use the sheet to confirm what happened and how often.
Do not treat a tally pattern as root-cause proof without follow-up analysis.

## Interpretation patterns

- Most frequent item: "[Check item] was tallied [count] times during [date range]."
- Pattern statement: "[Shift/location/date] shows a visible difference that needs follow-up."
- Next action: "Use the validated tallies for Pareto, stratification, or trend review."
- Caution wording: "The Check Sheet shows occurrence; it does not prove root cause."

## Common mistakes

Common mistakes include changing categories midstream, leaving ambiguous "other" notes, and mixing different collection periods.
Other review risks are missing source or date range, unclear observation definitions, and treating demo content as project evidence.

## Example

Use the Check Sheet PowerPoint template to structure baseline defect or error collection.

## Related methods

Related downstream methods include Pareto Chart, Histogram, and Stratification.

## Formula or logic notes

No first-slice formula is implemented for Check Sheet.
The template defines collection structure, while Python-generated charts consume validated data later.

## Python assist decision

Python assist is not normally needed for a Check Sheet template because the method is a template-native worksheet.
PowerPoint is enough when the team is designing or presenting a small tally sheet.
Use Python later only when summarizing large exported logs, validating many rows, or preparing high-rigor data-dependent evidence.

## Evidence levels and source notes

- Level 1 teaching/draft: PowerPoint edits are sufficient.
- Level 2 normal QCC project: preserve source, date range, collection rules, filters, and facilitator checklist evidence.
- Level 3 competition or management review: preserve source data, calculation table if totals are summarized, checklist, and the versioned template used.
- Level 4 audit or high-risk evidence: support data-dependent claims with a reproducible evidence package or another validated analysis path.

## Review checklist

- Confirm the observation definition is clear.
- Confirm categories are mutually understandable.
- Confirm the collection period is stated.
- Confirm the sheet is not treated as final calculation evidence by itself.
- Confirm the facilitator checklist verifies source, date range, key conclusion, and next action.
