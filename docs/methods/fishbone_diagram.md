# Fishbone Diagram

## Summary

A Fishbone Diagram organizes possible causes around a clearly stated effect.
In the first slice it is a template-guided method.

## QCC stage fit

Use it during Analyze Causes after the team has defined the problem and reviewed baseline evidence.

## When to use

Use Fishbone when the team needs to brainstorm and organize possible causes before verification.

## When not to use

Do not use it as proof that a listed cause is real.
Do not use it with a vague effect statement.

## Inputs required

Inputs include the effect statement, cause categories, brainstormed causes, and notes about how causes will be checked.

## Outputs produced

The method produces a cause map and a shortlist of causes that need evidence or verification.

## Procedure

1. Write one effect statement.
2. Select cause branches that fit the process.
3. Add possible causes under each branch.
4. Combine duplicates and clarify unclear causes.
5. Mark which causes need evidence.
6. Select follow-up checks or 5 Whys chains.

## Interpretation guidance

Treat the diagram as a structured hypothesis map.
Only causes supported by later evidence should drive countermeasures.

## Common mistakes

Common mistakes include listing symptoms as causes, mixing several effects, and voting on causes without verification.

## Example

Use the Fishbone Diagram template placeholder to organize suspected causes for a high-priority Pareto category.

## Related methods

Related upstream methods include Pareto Chart and 5W2H.
Related downstream methods include 5 Whys and countermeasure selection.

## Formula or logic notes

No first-slice formula is implemented for Fishbone Diagram.
The method records and organizes hypotheses.

## Review checklist

- Confirm the effect statement matches the project evidence.
- Confirm each cause is written clearly enough to verify.
- Confirm the diagram distinguishes suspected causes from verified causes.
- Confirm next verification actions are recorded.
