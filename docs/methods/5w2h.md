# 5W2H

## Summary

5W2H structures a QCC problem statement by asking what, why, where, when, who, how, and how much.
In the first slice it is a template-guided method.

## Purpose

5W2H turns a broad concern into a specific, reviewable problem statement.
It helps the team separate known facts from assumptions before analysis starts.

## QCC stage fit

Use it during Define Problem to make the project scope concrete before collecting or analyzing data.

## When to use

Use 5W2H when a problem statement is vague, too broad, or missing measurable context.

## When not to use

Do not use it as a substitute for actual measurement.
Do not force answers when the team has not yet collected enough facts.

## Inputs required

Inputs include the problem background, observed condition, affected process, time period, people or roles involved, current handling method, and measurable impact.

## Outputs produced

The method produces a structured problem definition that can guide Check Sheet design, Pareto analysis, and target setting.

## Procedure

1. Answer what is happening.
2. Explain why the issue matters.
3. Identify where it happens.
4. Identify when it happens.
5. Identify who is affected or involved.
6. Explain how it is detected or handled.
7. Quantify how much impact is known.

## PowerPoint template workflow

1. Open the 5W2H PowerPoint method template.
2. Review the completed DEMO EXAMPLE - not project evidence slide.
3. Replace the answer cells with project facts.
4. Copy the blank working slide or worksheet into the project deck.
5. Record the source, date range, known assumptions, and prepared-by/date.
6. Use the facilitator checklist before treating the statement as project scope.

PowerPoint is enough for teaching, draft, and normal 5W2H use.

## Blank working slide or worksheet

The blank working slide or worksheet should include the seven 5W2H fields, a source note, date range, key conclusion, next action, and prepared-by/date.
Keep unknown facts visible as gaps instead of filling them with guesses.

## Interpretation guidance

The output is useful when it narrows the project to a measurable problem.
If several answers remain unknown, use the gaps to plan data collection.

## Interpretation patterns

- Problem statement: "[What] occurs at [Where] during [When], affecting [Who], with [How much] impact."
- Scope decision: "This project focuses on [included area] and excludes [out-of-scope area]."
- Gap statement: "The team needs a Check Sheet or record review to confirm [unknown fact]."
- Caution wording: "This slide defines the problem; it does not prove root cause."

## Common mistakes

Common mistakes include writing opinions instead of observations, defining the solution inside the problem statement, and omitting measurable impact.
Other review risks are missing source or date range, mixing several problems in one statement, and treating demo content as project evidence.

## Example

Use the 5W2H PowerPoint template to turn a broad issue into a reviewable problem statement.

## Related methods

Related downstream methods include SIPOC, Check Sheet, and Pareto Chart.

## Formula or logic notes

No first-slice formula is implemented for 5W2H.
The method is a structured reasoning template.

## Python assist decision

Python assist is not normally needed for 5W2H because the method is a template-native reasoning worksheet.
PowerPoint is enough when the output is a training example, draft scope statement, or normal QCC project statement.
Use Python only later if the measurable impact comes from a large data extract or if high-rigor evidence needs a reproducible summary behind the number.

## Evidence levels and source notes

- Level 1 teaching/draft: PowerPoint edits are sufficient.
- Level 2 normal QCC project: preserve source, date range, assumptions, and facilitator checklist evidence.
- Level 3 competition or management review: preserve source data or source records, checklist, and the versioned template used.
- Level 4 audit or high-risk evidence: support data-dependent claims with a reproducible evidence package or another validated analysis path.

## Review checklist

- Confirm each W/H answer is specific.
- Confirm the problem statement does not prescribe the countermeasure.
- Confirm measurable impact is stated or marked as unknown.
- Confirm follow-up data collection is linked when facts are missing.
- Confirm the facilitator checklist verifies source, date range, key conclusion, and next action.
