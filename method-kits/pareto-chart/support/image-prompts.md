# Image Prompts: Pareto Chart

These prompts are support material for contributors.
They are conceptual only and do not create final project evidence.

## Concept visual prompt

### Purpose

Create a conceptual teaching visual for the Pareto Chart method.

### Use

training and explanation only.
not final project evidence.
not a quantitative chart.

### Prompt

Create a clean professional training visual showing the idea of vital few versus trivial many in quality defects.
Use simple generic bars without exact numerical values.
Avoid small text, precise axis labels, or calculated percentages.
Leave space for Markdown explanation beside or below the image.

### Negative constraints

Do not include exact data values.
Do not include fake percentages.
Do not include misleading axes.
Do not imply the image is project evidence.
Do not render detailed method instructions inside the image.
Keep detailed method instructions in Markdown.

## Good/bad comparison prompt

### Purpose

Create a conceptual teaching visual that compares a strong Pareto chart layout with a weak Pareto chart layout.

### Use

training and explanation only.
not final project evidence.
not a quantitative chart.

### Prompt

Create a clean professional training visual comparing a strong Pareto chart layout with a weak layout.
Show the strong side with descending bars, readable labels, source-note area, and a clear focus annotation.
Show the weak side with unsorted bars, missing source-note area, and cluttered labels.
Use generic shapes only and avoid exact numbers or calculated percentages.

### Negative constraints

Do not include exact data values.
Do not include fake percentages.
Do not include misleading axes.
Do not imply either side is project evidence.
Keep detailed method instructions in Markdown.

## Review checklist

| Check | Pass condition |
|---|---|
| Conceptual only | The image cannot be confused with final project evidence. |
| No exact numeric claims | The image avoids exact values, fake percentages, and precise axes. |
| Text-light | Detailed wording remains in Markdown. |
| Method-correct | The visual does not teach the wrong method idea. |
| Professional quality | The visual is suitable for a QCC training guide. |
