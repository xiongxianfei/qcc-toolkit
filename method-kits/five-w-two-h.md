## Summary

5W2H is a structured worksheet for clarifying a problem or defining an action.
It asks what, why, where, when, who, how, and how much so a broad concern becomes reviewable.

The method improves clarity.
It does not prove root cause, does not replace root-cause analysis, and does not prove that an action worked.

## QCC stage fit

Use 5W2H during Problem selection when the team needs to turn a broad concern into a specific, reviewable problem statement.
Use it during Countermeasure planning when the team needs to define an action clearly enough to assign, execute, and verify.

The handoff from problem-framing mode is usually to Check Sheet design, Pareto Chart, Flowchart / Process Map, or target setting.
The handoff from action-planning mode is usually to implementation, verification, standardization, or a monitoring plan.

## What question this method answers

What exactly is the problem or action, why does it matter, where and when does it apply, who is involved, how does it happen or get done, and how much impact, effort, resource, cost, or expected effect is involved?

## When to use

Use 5W2H when a problem statement is vague, broad, context-poor, or mixed with proposed solutions.
Use it when an action idea needs an owner, due date, dependencies, verification measure, expected evidence, and target or acceptance condition.
Use it when assumptions and constraints need to be visible before the team moves forward.

## When not to use

Do not use 5W2H as a substitute for measurement.
Do not use it as a substitute for root-cause analysis.
Do not use it to claim a countermeasure worked without verification evidence.
Do not use it to hide a solution inside the problem statement.

## Required inputs

- Problem background or selected action.
- Observed condition or intended countermeasure.
- Affected process, product, location, or scope.
- Time period or due date.
- People, roles, owner, or affected group.
- Current handling method or planned action method.
- Measurable impact, cost, effort, resource, or expected effect.
- Source, date range, assumptions, constraints, and review status.

## Output

The output depends on the mode.
Problem-framing mode produces a structured problem definition that can guide Check Sheet design, Pareto analysis, target setting, and follow-up collection.
Action-planning mode produces a structured action definition with owner, due date, dependencies, verification measure, expected evidence, target or acceptance condition, assumptions, and constraints.

The output is not a verified cause and not verified improvement.

## Manual chart or diagram recipe

This method produces a worksheet rather than a chart or diagram.
Create a table with one row for each W/H question and columns for answer, evidence or source, assumptions, owner, and review notes.
Use separate sections for problem-framing mode and action-planning mode so the team does not confuse problem definition with countermeasure definition.

## Worksheet purpose

The worksheet turns a broad concern into a specific, reviewable problem statement or turns a proposed action into an assignable plan.
It helps reviewers see what is known, what is assumed, what remains unknown, and what evidence will be used next.

## Problem-framing mode

Use Problem-framing mode to clarify what happened, why it matters, where it happened, when it happened, who is affected or involved, how it appears, and how much impact exists.

| Prompt | Problem-framing answer should clarify |
|---|---|
| What | What happened or what condition was observed. |
| Why | Why it matters to the customer, process, safety, cost, quality, delivery, or team goal. |
| Where | Where the condition appears in the process, product, location, or record. |
| When | When it appears, including date range, shift, frequency, timing, or trigger. |
| Who | Who is affected or involved, stated as roles or groups when possible. |
| How | How it appears, is detected, is currently handled, or is experienced. |
| How much | How much impact exists, or what measurable impact remains unknown. |

The problem statement should not prescribe the countermeasure.
If facts are missing, link the worksheet to follow-up collection.

## Action-planning mode

Use Action-planning mode to define what will be done, why it should be done, where it applies, when it is due, by whom it will be done, how it will be done, and how much cost, effort, resource, or expected effect is involved.

| Prompt | Action-planning answer should clarify |
|---|---|
| What | What will be done. |
| Why | Why this action is justified by the selected cause, target, or decision. |
| Where | Where the action applies. |
| When | When it starts, when it is due, and any timing constraints. |
| Who | By whom the action will be owned, supported, reviewed, or affected. |
| How | How the action will be executed. |
| How much | How much cost, effort, resource, capacity, risk, or expected effect is involved. |

Action-planning mode should include owner, due date, dependencies, verification measure, expected evidence, target or acceptance condition, assumptions, and constraints.
It improves clarity but does not prove that an action worked.

## Worksheet construction steps

1. Choose problem-framing mode or action-planning mode.
2. State the purpose of the worksheet.
3. Answer what, why, where, when, who, how, and how much.
4. Mark unknowns explicitly instead of guessing.
5. Record source, date range, assumptions, and constraints.
6. For problem framing, check that the problem statement does not define the solution.
7. For action planning, add owner, due date, dependencies, verification measure, expected evidence, and target or acceptance condition.
8. Review whether follow-up collection, root-cause analysis, verification, or standardization is needed next.

## Formatting standard

Use concise answers that a reviewer can trace to evidence.
Keep problem-framing and action-planning answers separate.
Mark unknown impact, unclear ownership, or unverified assumptions visibly.
Keep source/date/conclusion/next action visible.

## Required annotations

- Mode: problem-framing or action-planning.
- Source and date range.
- Scope, process, location, or affected group.
- Owner and due date when an action is defined.
- Dependencies.
- Verification measure.
- Expected evidence.
- Target or acceptance condition.
- Assumptions and constraints.
- Next method or next review action.

## Quality standards

A strong 5W2H worksheet has specific answers for each W/H prompt, visible source evidence, clear unknowns, and an explicit next handoff.
In problem-framing mode, it defines the problem without prescribing the countermeasure.
In action-planning mode, it defines ownership, timing, dependencies, verification, evidence, and acceptance conditions.

The worksheet is weak when it contains opinions without observations, omits measurable impact, mixes several problems, or treats action clarity as proof of improvement.

## Interpretation limits

Safe interpretation says the worksheet clarifies the problem or action.
Unsafe interpretation claims the worksheet proves root cause, proves improvement, or replaces verification.

Use 5W2H to make the next step reviewable.
Use root-cause analysis and verification evidence before claiming causes or results.

## Common mistakes

- Writing opinions instead of observations.
- Defining the solution inside the problem statement.
- Omitting measurable impact or marking unknowns as facts.
- Missing source or date range.
- Mixing several problems in one worksheet.
- Writing an action without owner, due date, dependencies, verification measure, expected evidence, or acceptance condition.
- Treating action framing as verified improvement.
- Treating demo content as project evidence.

## Evidence note

For project use, preserve source, date range, assumptions, constraints, facilitator checklist evidence, source records, and review status.
Problem-framing mode should preserve observed condition, affected process, time period, people or roles involved, current handling method, and measurable impact.
Action-planning mode should preserve owner, due date, dependencies, verification measure, expected evidence, target or acceptance condition, assumptions, constraints, and follow-up evidence.

Evidence level:

- Teaching or draft use can use a synthetic broad concern or action.
- Normal QCC project use should preserve the completed worksheet and review checklist.
- Formal review should preserve source data or records, facilitator checklist evidence, reviewer status, and follow-up verification evidence.
