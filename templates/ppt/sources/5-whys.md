## Presentation pattern

Use these source notes as the reviewable contract for the generated PowerPoint template.
The deck includes a data-entry slide with editable why-chain answers and evidence checks.
The final slide should show the cause chain, evidence at key links, and verification status for the proposed root cause.

## Method kit sections

### Purpose

Trace one focused problem or suspected cause through evidence-backed why links.

### QCC stage fit

Analyze Causes. Use after 5W2H, Pareto, or Fishbone narrows the focus.

### When to use

Use when the team can follow one cause chain and each answer can be checked.

### When not to use

Do not use it for broad multi-branch problems or as proof that the fifth answer is automatically root cause.

### Required inputs

Narrow starting problem, why-chain answers, evidence checks, candidate root cause, source, date range, and verification status.

### Completed demo example

The deck includes a completed DEMO EXAMPLE - not project evidence that shows a realistic why chain.

### Blank working slide or worksheet

The deck includes a blank working slide or worksheet users can copy into a QCC project deck.

### Interpretation patterns

Use patterns for cause chain, root-cause candidate, verification status, next action, and caution wording about unverified links.

### Common mistakes

Avoid branching into several problems, using opinions as answers, skipping evidence checks, missing source or date range, and demo content used as project evidence.

### Facilitator checklist

Check that the starting problem is narrow, each why follows from the previous answer, source and date range are shown, evidence checks are recorded, the key conclusion is written, and the next action is clear.

### Python assist decision

PowerPoint is enough for teaching, draft, and normal 5 Whys use.
Python is not normally needed unless verification evidence requires data summarization or high-rigor reproducible support.

### Evidence/source note

Record source, date range, evidence checks, assumptions, prepared-by/date, and whether evidence level is teaching/draft, normal QCC project, competition or management review, or audit/high-risk.
